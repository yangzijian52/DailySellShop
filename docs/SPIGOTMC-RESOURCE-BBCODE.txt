[CENTER][SIZE=7][B]DailySellShop Documentation[/B][/SIZE]
[SIZE=4]Version 2.0.0 - Paper 26.2[/SIZE][/CENTER]

[COLOR=#ff4d4d][B]Language notice:[/B][/COLOR] The SpigotMC resource page, documentation and support channel are English-only. Chinese-language support is not provided on SpigotMC.

[SIZE=5][B]Requirements[/B][/SIZE]
[LIST]
[*]Paper 26.2 only
[*]Java 25 or newer
[*]Vault
[*]A Vault-compatible economy plugin
[*]Geyser and Floodgate are optional and only required for Bedrock forms
[*]Plugin loading, Java GUIs, selling and purchasing were tested successfully on a live Paper 26.2 server
[/LIST]

[SIZE=5][B]Installation[/B][/SIZE]
[LIST=1]
[*]Stop the server.
[*]Install Vault and a compatible economy plugin.
[*]Install Geyser and Floodgate if Bedrock form support is required.
[*]Place DailySellShop-2.0.0.jar in the server plugins directory.
[*]Start the Paper 26.2 server.
[*]Review plugins/DailySellShop/SellShopconfig.yml and plugins/DailySellShop/shopconfig.yml.
[*]Run /dss reload or /ds reload after editing either configuration file.
[*]Test one sell transaction and one buy transaction before opening the server to players.
[/LIST]

[SIZE=5][B]Player Commands[/B][/SIZE]
[LIST]
[*][B]/ds[/B] - open the rotating sell shop
[*][B]/dss[/B] - open the permanent buy shop
[*][B]/dss search <name or Material>[/B] - search the buy catalog
[*][B]/dss help[/B] - display command help
[/LIST]

[SIZE=5][B]Admin Commands[/B][/SIZE]
[LIST]
[*][B]/ds reload[/B] - reload both configuration files and refresh the sell catalog
[*][B]/dss reload[/B] - reload both configuration files and refresh both shops
[*][B]/dss open <player>[/B] - open the buy shop for an online player
[/LIST]
Reloading cancels every pending custom-quantity transaction.

[SIZE=5][B]Aliases[/B][/SIZE]
No command aliases are registered. [B]/ds[/B] and [B]/dss[/B] are separate commands with different purposes.

[SIZE=5][B]Permissions[/B][/SIZE]
[LIST]
[*][B]dailysell.admin[/B] - use /ds reload - default: OP
[*][B]dailysellshop.shop.use[/B] - open and search the buy shop - default: all players
[*][B]dailysellshop.shop.buy[/B] - purchase regular products - default: all players
[*][B]dailysellshop.shop.spawnegg[/B] - purchase enabled spawn eggs - default: all players
[*][B]dailysellshop.shop.admin[/B] - use /dss open and /dss reload - default: OP
[*]A regular product or spawn egg may define an additional permission in shopconfig.yml.
[/LIST]

[SIZE=5][B]Sell Shop Flow[/B][/SIZE]
[LIST=1]
[*]The server selects products from SellShopconfig.yml according to the DAILY or HOURLY refresh mode.
[*]The player opens /ds and selects a displayed product.
[*]The plugin counts only matching items in normal inventory storage.
[*]The remaining personal sell limit determines the maximum accepted amount.
[*]Matching items are removed and the Vault economy deposit is attempted.
[*]On success, the personal limit is updated. If the deposit fails, the inventory snapshot is restored.
[/LIST]

[SIZE=5][B]Buy Shop Flow[/B][/SIZE]
[LIST=1]
[*]The player opens /dss and selects one of ten categories, or uses /dss search.
[*]The player selects a product and chooses a preset or custom quantity.
[*]The confirmation screen shows quantity, total price, balance and estimated inventory space.
[*]The plugin revalidates the product, category, permission, amount, balance and capacity.
[*]Money is withdrawn before items are delivered.
[*]If delivery fails, the inventory is restored and the payment is refunded.
[*]A successful purchase is written to the configured UTF-8 transaction log.
[/LIST]

[SIZE=5][B]Custom Quantity Rules[/B][/SIZE]
[LIST]
[*]Default maximum: 2,304 items
[*]Default input timeout: 60 seconds
[*]Java players enter the number in chat; Bedrock players use a numeric form
[*]One invalid input retry is allowed; the original timeout is not restarted
[*]Input is cancelled after excessive movement, world change, death, logout, timeout or plugin reload
[*]Entering cancel cancels the transaction; the default Chinese cancellation keyword is also recognized
[*]If the full amount does not fit, only the amount that fits is purchased
[/LIST]

[SIZE=5][B]Configuration Files[/B][/SIZE]
[LIST]
[*][B]SellShopconfig.yml[/B] - refresh mode, sell limits, /ds GUI, product names and base prices for 1,477 products
[*][B]shopconfig.yml[/B] - /dss categories, GUIs, purchase rules, price overrides, sounds, logging and 88 spawn eggs
[*][B]creative-order.yml[/B] - internal Mojang 26.2 category/order catalog; not intended for normal editing
[/LIST]
Both player-editable files contain detailed Chinese comments and must remain UTF-8. Use spaces for YAML indentation; do not use tabs.

[SIZE=5][B]Common Sell Configuration[/B][/SIZE]
[CODE]timezone: "Asia/Shanghai"
refresh-mode: "HOURLY" # DAILY or HOURLY

random-rotation:
  enabled: true
  amount: 5

hourly-settings:
  multiplier: 0.5

menu:
  size: 54
  sell-slots: [20, 21, 22, 23, 24][/CODE]

[SIZE=5][B]Common Buy Configuration[/B][/SIZE]
[CODE]settings:
  buy-price-multiplier: 1.0
  custom-amount-max: 2304
  custom-input-timeout-seconds: 60
  custom-input-cancel-distance: 2.0
  bedrock-page-size: 20

logging:
  enabled: true
  console: false
  file: "logs/purchases.log"[/CODE]

[SIZE=5][B]Regular Product Override[/B][/SIZE]
[CODE]product-overrides:
  DIAMOND:
    enabled: true
    name: "&bDiamond"
    price: 7.50
    category: raw-materials
    weight: 100
    permission: ""[/CODE]
The product key must exactly match an item key in SellShopconfig.yml. Lower weight values appear earlier. A non-empty permission hides and blocks the product for players without that permission.

[SIZE=5][B]Spawn Egg Configuration[/B][/SIZE]
[CODE]spawn-eggs:
  COW_SPAWN_EGG:
    enabled: true
    name: "&fCow Spawn Egg"
    price: 120.00
    weight: 100
    max-amount: 2304
    require-permission: false
    permission: "dailysellshop.shop.spawnegg.cow"[/CODE]
Boss, unsafe and special entities should remain disabled unless the server owner has reviewed the gameplay and performance impact.

[SIZE=5][B]Configurable Menu Actions[/B][/SIZE]
[CODE]commands:
  - "[command] quickmenu open"
  - "[console] say {player} opened the shop"
  - "[message] &aWelcome to the shop"
  - "[sound] UI_BUTTON_CLICK"
  - "[close]"[/CODE]
[LIST]
[*][B][command][/B] runs as the player
[*][B][console][/B] runs from the server console
[*][B][message][/B] sends colored text to the player
[*][B][sound][/B] plays a configured Bukkit sound
[*][B][close][/B] closes the menu
[*][B][op][/B] temporarily grants operator status and must only be used in trusted configuration files
[/LIST]
Common placeholders include {player}, {balance}, {item}, {material}, {price}, {amount}, {total}, {slots}, {capacity}, {count}, {page} and {pages}.

[SIZE=5][B]Data Storage and Backups[/B][/SIZE]
[LIST]
[*]No external database is used.
[*]Configuration is stored in plugins/DailySellShop/SellShopconfig.yml and shopconfig.yml.
[*]Purchase audit records are appended to plugins/DailySellShop/logs/purchases.log by default.
[*]Per-player sell progress is stored in memory for the active refresh period and resets after a full server restart.
[*]Back up both YAML files before upgrades. Back up purchases.log if transaction history must be retained.
[*]The plugin never overwrites existing configuration files during startup.
[/LIST]

[SIZE=5][B]Upgrading from 1.x[/B][/SIZE]
[LIST=1]
[*]Stop the server.
[*]Back up the complete plugins/DailySellShop directory.
[*]Replace the old jar with DailySellShop-2.0.0.jar.
[*]Migrate custom settings from config.yml to SellShopconfig.yml.
[*]Manually merge the 2.0.0 product, potion, enchanted-book and price nodes.
[*]Review the new shopconfig.yml and configure spawn eggs, menus and purchase rules.
[*]Start the server and verify the loaded product and spawn-egg counts.
[*]Test one /ds sale and one /dss purchase.
[/LIST]
[COLOR=#ff4d4d][B]Important:[/B][/COLOR] Renaming config.yml does not merge the new 2.0.0 catalog. Existing configuration files are not automatically updated.

[SIZE=5][B]Known Limits[/B][/SIZE]
[LIST]
[*]Paper 26.2 and Java 25 are required; other server implementations and older versions are not supported.
[*]Transactions require Vault and an active Vault-compatible economy provider.
[*]There is no built-in database, stock system or persistent daily purchase-limit system.
[*]Per-player sell progress is not persisted across a full server restart.
[*]The default configurations and most administrator-facing messages are Chinese.
[*]The Java transaction flow was live-tested; Bedrock forms were not included in the reported live-server validation.
[*]Large custom quantities are capped by configuration and available inventory capacity.
[/LIST]
